Day 1 (16/02/2026) – Project Setup

In the initial meeting, the team discussed the project requirements and overall objectives. 
A GitHub repository was created to manage the project and allow team collaboration through version control. 
Team members were also assigned specific roles and responsibilities to ensure clear task ownership and efficient project management.

Day 2 (17/02/2026) – Task Planning

The project assignment was broken down into smaller, manageable tasks. 
These tasks were then distributed among the team members based on their roles and strengths. 
This helped organise the workflow and establish a clear development roadmap for the project.

Day 3 (18/02/2026) – Environment Setup

All necessary development tools and software were installed and configured to prepare for development. 
This included setting up .NET MAUI for application development and XAMPP for the backend/database environment. 
Ensuring the correct setup allowed the team to begin development smoothly.

Day 4 (19/02/2026) – UI Design

Initial user interface designs were created in Figma for all six pages of the application. 
These designs focused on layout, navigation, and overall user experience. 
The purpose was to create a visual representation of the application before development began.

Day 5 (20/02/2026) – Client Presentation

The Figma designs were presented to the client to gather feedback and ensure the design aligned with their expectations. 
The client provided feedback and suggestions, which will be used to refine the design before moving forward with development.
